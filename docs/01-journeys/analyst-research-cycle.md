# 旅程：分析师 - 完整研究循环

## 元信息
- 角色：独立行业分析师 林然（自由职业，4 年行业研究经验）
- 意图：用思考画布替代原有 Notion + ChatGPT 工作流，研究"中国新茶饮 2026 出海机会"，最终产出 8000 字深度报告
- 优先级：P0（核心路径，验证 PRD §2.1 的三个核心假设）
- 来源：PRD §2.1（假设验证）、§2.2（用户画像）、§4（四个核心动作）

## 前置状态
- Electron 应用首次启动后已完成配置：
  - LLM baseURL：`https://api.deepseek.com/v1`
  - 模型：`deepseek-reasoner`（用户选了带思考模式的模型）
  - API Key：已填入并保存到 OS keychain
  - 思考模式开关：**开启**（用户在设置页打开）
  - 已勾选过一次性隐私提示
- 画布状态：空白，4000×4000 可见区域，视野中心位于坐标 (0, 0)

## Happy Path

### 阶段 A：第一次种节点 + 主问题对话（约 5 分钟）

1. **林然双击画布中央偏上的空白处**（坐标约 (0, -200)）
   - 一个对话节点在双击位置生成（白底、细灰色边框、左上角对话气泡图标）
   - 节点宽度 380px，高度自适应（首次仅有输入框，约 80px 高）
   - 节点立即获得活跃态（淡蓝边框、103% 放大）
   - 输入框获得焦点，光标闪烁

2. **林然输入第一个问题**：
   ```
   我在写一份"中国新茶饮 2026 出海机会"的深度报告，目标读者是
   投资人和品牌方。先帮我梳理：当前中国新茶饮品牌出海的主要阻力
   有哪些？请按重要性排序，并各举 1-2 个真实案例。
   ```
   - 按 Enter 提交

3. **AI 流式回复开始**
   - 节点上方出现一个浅灰色折叠区域，标题 "💭 思考中..."（流式期间展开）
     - 显示 reasoning 内容，约 12 行，灰色小字
     - 思考片段示例："用户在写报告，要的是结构化分析…需要从供应链、本地化、监管、品牌力四个维度切入…要给案例支撑而不是泛泛而谈…"
   - 思考完成后，下方出现 assistant 消息流式输出：
     ```
     四个主要阻力，按重要性排序：

     1. 供应链与原料本地化（最大阻力）
        茶叶、奶基底、糖浆等核心原料的跨国供应链未成熟。蜜雪冰城在
        印尼通过自建基地缓解，但 2024 年泰国门店因原料断供曾大面积
        闭店。

     2. 口味本地化与品牌叙事
        霸王茶姬主打"东方茶"叙事在马来西亚奏效，但在欧美遇冷——
        消费者对"茶饮"这个品类本身缺乏中式语境。

     3. 监管与食品标准
        欧盟食品添加剂标准比中国严格，喜茶 2023 年试入德国时部分
        SKU 因色素问题被迫下架重做。

     4. 单店模型的本地化
        国内"小店密集 + 外卖"模型在东南亚部分有效，在欧美高人工
        成本下完全不可复制——必须做大店 + 体验式消费。
     ```
   - 思考区域**自动折叠**为一行灰色摘要 "💭 思考过程（点击展开 12 行）"，可点开
   - 节点高度增长到约 480px

4. **节点标题自动生成**（不可见过程）
   - 现在累计 2 条消息（user + assistant），未达 3 条阈值，标题暂未生成
   - 节点折叠按钮旁显示占位标题 "新节点"

5. **林然继续追问**：
   ```
   第 2 点"品牌叙事在欧美遇冷"——你是说茶饮品类天然不适合欧美？
   还是说叙事策略要调整？这个判断对我的报告很关键，请给我
   更细致的分析和反例。
   ```
   - AI 思考片段："用户挑战了一个简化的判断…要给出更精细的分层…"
   - AI 回复约 240 字，区分了"奶茶""果茶""纯茶"在欧美三种不同的接受度，并举出 Boba Guys（旧金山）的成功案例作为反例

6. **节点标题更新**（第 3 条消息后触发）
   - 后台调起一次轻量 LLM 调用："用 8-15 字概括这段对话核心"
   - 标题生成为 "新茶饮出海四阻力辨析"
   - 节点折叠按钮上方出现这个标题

### 阶段 B：分支并行探索（约 15 分钟）

7. **林然鼠标悬停在 AI 第一条回复（4 阻力概述）上**
   - 悬停超过 200ms 后，回复右下角浮出一个 "↳ 从这里分支" 按钮（仅在悬停时显示）

8. **林然点击 "从这里分支"**
   - 一个新的对话节点在原节点右侧 + 60px 偏移处（坐标约 (440, -200)）生成
   - 一条细灰色直线连接两节点（父→子方向，无箭头装饰）
   - 子节点继承父节点截至该回复的全部对话历史（即 user 第一问 + assistant 第一答，共 2 条消息）
   - 子节点立即获得焦点
   - 父节点退出活跃态（恢复正常透明度）

9. **林然在子节点中输入**：
   ```
   聚焦讲讲第 1 点供应链问题。我需要：(a) 各核心原料的来源国和
   替代方案、(b) 已出海的头部品牌（蜜雪、霸王茶姬、喜茶、奈雪）
   各自的供应链解法、(c) 一个明显的供应链失败案例及教训。
   ```
   - 这条新对话只属于子节点；父节点对话不变

10. **AI 思考 + 流式回复**（细节略，约 600 字结构化回答）
    - 子节点高度增长到约 520px

11. **林然回到父节点**：点击父节点
    - 父节点恢复活跃态，子节点退出活跃态
    - 输入框聚焦到父节点

12. **林然在父节点（不是子节点）继续提问**：
    ```
    现在换一个维度：监管层面。欧盟、东南亚、北美三个市场的食品
    监管差异有多大？哪些是出海前必须做的合规预备？
    ```
    - 这条对话进入父节点，与子节点（供应链子话题）并行
    - 父节点积累到 5 条消息

13. **林然从父节点最新这条 AI 回复再分支一次**
    - 第二个子节点生成，位于父节点右侧、第一个子节点下方（坐标约 (440, 200)）
    - 同样继承到该 AI 回复为止的父节点对话
    - 林然继续追问监管细节

14. **此时画布状态**：
    - 1 个父节点（标题 "新茶饮出海四阻力辨析"）
    - 2 个并行子节点：标题分别为 "供应链原料替代方案" 和 "三市场监管合规要点"
    - 父节点向下还有一条边连到刚刚的第三个分支……
    - 林然在第三个分支里聊了 4 轮关于"品牌叙事"

### 阶段 C：第一次提炼（约 3 分钟）

15. **林然意识到三个子分支已经成熟，需要综合**
    - 用 Shift+点击依次选中三个子节点（父节点不选）
    - 三个节点边框变为淡紫色（多选状态）
    - 画布顶部工具栏出现 "提炼 (3)" 按钮，同时右键菜单中也出现 "提炼" 项

16. **林然在工具栏点击 "提炼 (3)"**
    - 在被选中节点的几何中心位置（约 (440, 0)）浮出一个轻量 popover（不是 modal）
    - popover 内容：
      ```
      这次提炼要回答什么问题？
      [ 输入框，placeholder："留空表示综合性提炼"]
      [取消]  [提炼]
      ```

17. **林然输入**：
    ```
    供应链、监管、品牌叙事三方面问题如何整体影响一个新茶饮品牌
    选择出海首站？给出一个决策框架。
    ```
    - 点击 "提炼"
    - popover 关闭

18. **AI 提炼调用进行**（这是一次独立的写作任务，AI 不知道画布存在）
    - popover 位置出现一个临时占位（"正在提炼…" + 旋转图标）
    - 思考模式开启时，提炼调用也会带 reasoning 内容
    - 流式输出（约 380 字）：
      ```
      ## 决策框架：新茶饮出海首站选择

      **核心结论**：首站选择应基于"供应链可达性 × 监管熔断风险 ×
      品牌叙事适配"的最弱环节决定，而非单一维度的最优。

      **关键论据**：
      - 供应链不可达 = 直接死刑（蜜雪泰国闭店）
      - 监管不通过 = 资源沉没（喜茶德国 SKU 被迫重做）
      - 叙事不适配 = 增长上限低（茶饮品类在欧美 vs 东南亚的天然
        接受度差异）

      **决策建议**：
      1. 第一阶段：东南亚（华人圈层 + 供应链距离短 + 监管相对宽松）
      2. 第二阶段：基于第一阶段供应链能力外溢到中东 / 北美
      3. 欧盟应作为长期目标，不作为首战场

      **未解决/待验证**：
      - 中东市场（非传统茶饮地区）的实际数据缺失
      - 蜜雪 vs 霸王茶姬的供应链投入回报比未量化
      - "本地化大店模型"在欧美的最低盈利单店量未敲定

      **下一步建议**：
      - 找蜜雪/霸王茶姬 2024 年报对比供应链 capex
      - 调研 Boba Guys 在旧金山的单店营收和员工模型
      ```

19. **提炼节点生成**
    - 节点视觉：浅米色背景、稍粗边框、左上角棱形图标（明显区别于对话节点）
    - 节点位置：三个子节点的几何中心 + 偏移避让（坐标约 (440, 50)，自动避开重叠）
    - 三条细灰色直线分别从三个原节点连到提炼节点（多父结构）
    - 原节点不动、保持可见、可继续对话
    - 提炼节点的标题（首次自动生成）：「出海首站决策框架」

### 阶段 D：在提炼节点上攀升（约 10 分钟）

20. **林然点击提炼节点**
    - 提炼节点变为活跃态
    - 输入框聚焦
    - 输入框旁有一个小问号图标（hover 显示提示："此节点继续对话只用提炼内容作为上下文，不带入原节点完整对话"）

21. **林然输入新对话**：
    ```
    基于这个框架，如果我是一个 5 亿融资规模的新锐品牌（产品力强
    但供应链经验弱），首站推荐是什么？预算分配建议？
    ```

22. **AI 上下文（产品层组装的，AI 不知道）**：
    - **只包含**：提炼节点的纲要内容（约 380 字） + 林然这条新对话
    - **不包含**：三个原子节点的完整对话内容（PRD §7.4）

23. **AI 回复**（约 280 字）
    - 推荐马来西亚为首站（理由：华人 + 清真供应链共存 + 蜜雪已建立的供应链基础设施可借用）
    - 给了一个 5 亿预算的分配建议（供应链 30%，门店 35%，市场 20%，组织 15%）

24. **林然此时画布状态**：
    - 拓扑：1 父对话 → 3 子对话 → 1 提炼（多父）→ 该提炼节点已有 4 条消息
    - 林然实际感觉："我的思考真的在攀升——上面这个节点比下面三个加起来都更接近报告需要的结论"

### 阶段 E：再次循环（攀升）

25. **林然又分别从提炼节点的 AI 回复中分支了 2 次**：
    - 一次问"供应链 30% 预算的具体投向"
    - 一次问"组织 15% 预算下，团队如何配置"

26. **新分支节点是从提炼节点派生的对话节点**（不是提炼节点）
    - 它们继承提炼节点的纲要 + 该节点已有对话作为上下文
    - 视觉是普通对话节点（白底）

27. **第二天，林然回到画布**
    - 应用打开后视野自动恢复到上次离开时的位置
    - 所有节点、连线、折叠态完全恢复
    - 林然找到提炼节点 "出海首站决策框架"，继续推进——他不需要重新解释上下文

### 阶段 F：信号已达成（产出层面）
- 林然把提炼节点的内容直接复制到 Notion，作为报告"出海战略章节"的骨架
- PRD §10 三个指标在这一次 session 中都达成：
  - **分支率**：1（创建了 5 个分支）
  - **提炼率**：3/5 = 60%（三个子分支都被提炼）
  - **回流引用**：在分支 25-26 中，林然显式基于提炼节点的纲要继续推理

## Sad Path 1：林然在阶段 A 步骤 5 时网络抖动
- 步骤 5 的 AI 流式输出到约 60% 时网络中断
- 节点内出现红色错误气泡："网络中断，已收到内容已保留" + "重试" 按钮
- 已收到的部分流式 token **保留**在节点中（约 140 字）
- 重试按钮点击后：从中断处重新发送相同请求（不是续传，是重新调用），AI 重新输出一条完整回复
- 旧的不完整内容用淡灰色保留，新内容覆盖在下方（用户可手动删除旧的，MVP 期不自动清理）
- *对应 Q3-2 选项 A*

## Sad Path 2：阶段 E 后林然继续深聊提炼节点 20 轮，触发上下文超长
- 在提炼节点的某次新对话提交时，前端先做 token 估算（粗略基于字符数 × 0.6）
- 估算超过模型 context 上限的 80%（如 deepseek-reasoner 约 64K，阈值约 51K）
- 提交被前端拦截（不发送到 LLM）
- 节点内出现黄色提示气泡：
  > 当前对话已较长，继续可能超出模型上下文。
  > 建议：选中此节点 + 相关节点做一次"提炼"，基于提炼内容再继续。
  > [仍然发送]  [关闭提示]
- 用户点击 "仍然发送" 仍可强行发送（产品不强制阻断）；
- 若发送后 LLM 返回上下文超长错误，则回退为 Sad Path 1 风格的红色错误气泡，但提示文案改为 "上下文超长，建议先提炼"
- *对应 Q3-3 选项 A*

## 边界
- 100 节点上限：达到时新建节点会被允许，但顶部 toast 提示性能可能下降（Q5-1 选 B）
- 缩放 < 50%：所有节点变为色块 + 标题（超折叠态）
- 节点重叠允许，连线自动跟随
- 提炼节点也是活节点，可被再分支（Q8-1）
