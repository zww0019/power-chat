# 旅程：分析师 - 画布编辑、撤销、整理

## 元信息
- 角色：独立行业分析师 林然（同 journey-1）
- 意图：在画布上做日常编辑动作（移动节点、折叠/展开、删除、撤销、重做、画布缩放/平移），以及画布累积到 30+ 节点后的整理体验
- 优先级：P1（不直接验证 PRD 三个核心假设，但决定"画布是认知资产"承诺能否兑现——丢数据 = 产品死亡）
- 来源：Q2-1 ~ Q2-5、Q4-3、Q4-7、Q5-1 ~ Q5-4、Q7-4、PRD §6（画布物理）

## 前置状态
- 应用已运行良好，配置无问题
- 画布上已有 32 个节点（24 对话 + 8 提炼），分布在约 5000×5000 的逻辑空间
- 林然的视野当前缩放 80%，居中位置约 (1200, 800)
- 当前活跃节点 N17（一个对话节点，对话进行了 4 轮）

## Happy Path A：日常编辑流（约 10 分钟）

1. **林然把节点 N17 拖到画布另一个位置**
   - 鼠标按住节点 header 区（不是输入框，不是消息区），开始拖动
   - 节点跟随鼠标实时移动；连接到 N17 的边（父边 + 提炼子边）自动跟随
   - 拖动过程中，与目标位置邻近的其他节点**不自动避让**（允许重叠）
   - 释放鼠标在新位置 (1500, 1100)
   - 拖拽期间整个动作被记录为 **1 步 undo**（Q2-5），不是逐像素

2. **林然按 Cmd+Z（macOS）**
   - 节点 N17 平滑动画回到拖拽前位置 (1200, 800)，约 200ms 动画
   - 系统底部短暂显示一个轻量 toast："已撤销：移动节点"

3. **林然按 Cmd+Shift+Z（重做）**
   - 节点回到 (1500, 1100)
   - toast："已重做：移动节点"

4. **林然手动折叠某个聊太长的节点**
   - 点击节点 header 上的"−"按钮（或快捷键 Cmd+`）
   - 节点收缩为紧凑卡片：
     - 高度从 720px 压到约 60px
     - 显示标题 "霸王茶姬马来西亚单店模型" + 一行预览（最近一条 AI 回复的前 35 字）
     - 折叠态不能直接对话（输入框消失）
     - 节点不再是活跃态（边框恢复灰色）
   - 此动作 = 1 步 undo

5. **林然在折叠态节点上单击**
   - 节点展开 + 立即获得活跃态 + 输入框聚焦
   - 这个"展开"动作不进入 undo 栈（因为它是查看动作，不是结构动作）

6. **林然按 Cmd+滚轮缩小到 40%**
   - 缩放范围 25%-200%，40% < 50% 阈值
   - 所有节点进入**超折叠态**：
     - 对话节点显示为白色色块（30×40px），上面叠一行截短标题（最多 8 字）
     - 提炼节点显示为浅米色色块（视觉权重略高于对话节点）
     - 边变得更细
     - 输入框、对话消息等内部细节全部隐藏
   - 林然能从 "高空" 看到整个 32 节点拓扑：
     - 左下角是初始的"四阻力辨析"节点
     - 中部有 3 个并列的提炼节点（深米色）
     - 右上角是最新的几个深度对话
   - 林然心智："像在看一份地图，能直观看出我研究的攀升路径"

7. **林然 Cmd+滚轮放大回 100%**
   - 节点恢复完全显示

8. **林然按住空白处拖拽**：画布平移
   - 视野跟随鼠标平移，所有节点相对位置不变
   - 不进入 undo 栈

## Happy Path B：删除节点 + 撤销

9. **林然发现一个早期节点 N3 内容已经过时（早期试探性的对话）**
   - N3 有 1 个父边（连到 N1）+ 1 个被提炼子边（连到提炼节点 R2）

10. **林然右键 N3，菜单出现：**
    - 折叠 / 展开
    - 删除节点
    - 复制内容到剪贴板
    - （选中多个时还会有"提炼 (N)"，单选时无）

11. **林然点击 "删除节点"**
    - 弹出二次确认（避免误删）：
      > 删除该节点？（无法在跨会话后撤销，但本次会话内可 Cmd+Z）
      > [取消]  [删除]
    - 林然确认删除

12. **删除生效（按 Q2-3 选项 B：保留但断开）**
    - N3 节点从画布消失（淡出动画 200ms）
    - **不删除** N3 的子孙节点（如果有），但**断开** N3 与它们之间的边
    - **不删除** R2，但断开 R2 与 N3 的边（R2 仍保留与其他原节点的边，仍是合法的多父提炼节点）
    - 视觉上：原来 R2 周围有 3 条边，现在剩 2 条
    - 此动作 = 1 步 undo

13. **林然意识到删错了，按 Cmd+Z**
    - N3 重新出现（带原内容、原位置）
    - 与 N1 的父边、与 R2 的子边都恢复
    - toast："已撤销：删除节点"

## Happy Path C：撤销栈饱和

14. 假设林然之前一通乱试，做了 50 次操作
15. 第 51 次操作（移动某节点）压入栈时，最早的那次操作出栈（FIFO）
    - 用户感知不到（除非他要撤销超过 50 步）
    - 跨会话不保留 undo 历史（Q2-4）

16. 林然关闭并重新打开应用：
    - 画布完全恢复（节点、位置、对话、折叠态、视野）
    - **undo 栈清空**——他不能再撤销关闭前的操作
    - 这个限制由 Stage 0 决定，UI 不专门提示（但首次启动后会有一次性 toast：欢迎回来 / 上次会话已保存）

## Sad Path 1：在 AI 流式输出过程中操作节点（Q4-3 选 B：允许）

17. 林然在节点 N20 提交一个新问题，AI 开始流式输出（约 8 秒过程）
18. 流式进行到一半，林然意识到要把这个节点挪到画布右侧整理一下
19. 他直接拖动 N20——**允许**：
    - 节点位置跟随鼠标移动
    - 流式输出继续在后台进行（assistant 消息仍在累积 token）
    - 拖动结束时，新位置正常保存
20. 流式完成，林然看到完整回复
21. 流式期间他还可以：折叠 N20（流式继续；折叠态卡片下方显示"AI 正在回复…"指示器）；切换到别的节点继续读
22. 但他**不能**：删除 N20（删除按钮在流式期间禁用，hover 显示 "AI 正在回复，请先停止生成"）

## Sad Path 2：节点超过 100 软上限

23. 林然这周的研究持续投入，画布累计到 100 个节点
24. 他双击空白创建第 101 个节点
    - 节点正常创建
    - 顶部 toast 提示（5 秒后自动消失）：
      > 当前画布已有 100+ 节点，性能可能下降。
      > 建议：用提炼整合并删除已废弃节点。
    - 不阻断创建（Q5-1 选 B）

25. 林然此时画布渲染开始略卡（拖拽偶有掉帧）
    - 视野外的节点已被虚拟化（不渲染 DOM）
    - 但缩到俯瞰视角（< 50%）时所有节点都要渲染色块——这个量级暂可接受

## Sad Path 3：误拖动节点出视野

26. 林然不小心把一个节点拖到了非常远的位置（坐标 (15000, -8000)）
27. 节点暂时从视野消失
28. **恢复方式**：
    - Cmd+Z 撤销拖拽
    - 或者：缩到极小（25%）总能看到全部节点位置，找到后双击节点实现"飞入"（动画过渡到该节点居中、缩放到 100%）
29. MVP 不提供"找到我的节点"的搜索功能（PRD §8 排除）

## Sad Path 4：键盘快捷键失灵 / 误触

30. 林然在某个节点输入框中按 Cmd+Z
    - 此时 Cmd+Z 应作用于**输入框文本**（撤销刚输入的字），而**不是**画布动作
    - 仅当焦点不在任何输入框时，Cmd+Z 才作用于画布
31. 林然在画布空白处按 Tab 键
    - 切换到最近编辑过的节点（Q7-4 基本键盘可达）
    - Tab 反复按 → 在 N 个最近节点间循环

## 边界
- 拖动开始位置在节点 header 才生效，避免与消息区域文本选中冲突
- 折叠态节点也可拖动
- 拖动到画布边缘自动平移视野（连续移动时）
- 删除提炼节点时：原节点的边自动断开（同对话节点删除规则）
- Tab 切换的"最近"定义：按节点 last_focused_at 倒序，无最近记录则按创建时间

## 待确认（汇入 00-gaps.md 延伸）
- [ ] 流式期间的"AI 正在回复"指示器，在折叠态卡片上的具体视觉？倾向：标题旁有个小 spinner
- [ ] 节点头部的折叠/展开按钮 vs 整体点击展开——折叠态点击节点本体即展开（步骤 5），展开态需要点 "−" 按钮才折叠。这种不对称是否会让用户困惑？倾向 Stage 3 原型阶段实测
