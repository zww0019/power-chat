## 9. Stage 1 衍生缝隙（P2，全部按倾向落定）

> 这些是写用户旅程脚本时才暴露的细节，全部按"倾向"通过，不阻塞下游阶段。如有原型阶段实测后需要修正的，回到本节更新。

### A. 视觉与交互细节

- **A1** 流式期间在折叠态卡片上的"AI 正在回复"指示器 → **标题旁加小 spinner**
- **A2** 节点折叠/展开的对称性（折叠态点本体即展开 vs 展开态需点 "−" 按钮才折叠） → **Stage 3 原型实测决定**
- **A3** 双击折叠态节点是否同时展开 + 自动滚到最新消息 → **是**
- **A4** 节点拖动是否有"贴吸"或"对齐辅助线" → **MVP 不做**
- **A5** Tab 键切换节点的视觉过渡 → **平滑视野滚动**

### B. 多选与编辑动作

- **B1** 多选节点的方式 → **MVP 只支持 Shift+点击，不做框选**
- **B2** 撤销删除节点后位置是否恢复到删除前 → **是**（位置纳入 undo 状态）
- **B3** 同时删除多个节点 → **支持，作为 1 步 undo**

### C. AI / Token 相关

- **C1** 思考模式开启但模型不支持 reasoning 时 → **静默降级，不显示思考区域，不弹提示**
- **C2** "继续生成"恢复策略对 AI 连贯性的影响 → **Stage 3 原型实测**
- **C3** Token 估算系数 → **单一保守系数：中文 ×0.7 / 英文 ×0.4，不做模型适配**
- **C4** 上下文超长拦截阈值是否开放给用户调节 → **不开放，固定 80%**

### D. 配置 & Provider

- **D1** 设置页"保存并测试"调用哪个端点 → **`/v1/models`**
- **D2** 切换 provider 后已有节点的对话历史是否标记"由模型 X 生成" → **MVP 不做，二阶段再考虑**

### E. 应用启动与性能

- **E1** 第二天打开应用时视野自动恢复的语义 → **恢复到关闭时的视野位置**
- **E2** 提炼 popover 在被选中节点距离很远时 → **保证 popover 在当前视野内，不跟随几何中心跑出视野**
- **E3** 节点上限的硬拒绝阈值 → **MVP 不设硬拒绝**
