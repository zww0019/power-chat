# 项目知识库使用指南

## 重要：业务知识管理

本项目使用 `md/` 文件夹管理业务知识，纳入 Git 版本控制。

**在处理任何业务相关任务前，按需通过 project-knowledge-curator agent 查询项目知识库。**

### 知识目录结构

```
md/
├── 00-总览/           # 业务全景、模块关系
├── 01-业务实体/       # 实体定义、状态流转
├── 02-业务规则/       # 业务规则、约束条件
├── 03-架构设计/       # 模块划分、接口设计
├── 04-技术决策/       # 技术选型、架构决策
└── 05-变更记录/       # 日常变更记录
```

### 知识分层

- **L1 核心规则**: 必须严格遵守（多次引用且未修改）
- **L2 约束条件**: 不可绕过（已确认的约束）
- **L3 实现参考**: 可调整（初次记录或推断）

### 相关命令

- `/pkquery [关键词]` - 查询项目知识库
- `/pinit-md` - 初始化项目知识库
